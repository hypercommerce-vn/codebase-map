Vấn đề: AI Coding tốn quá nhiều token để khám phá code
Phần lớn token bị tiêu hao cho việc "tìm hiểu" thay vì "làm việc"

Khi AI coding tool (Claude, Cursor, Copilot) làm việc với codebase lớn, phần tiêu tốn nhất không phải là viết code — mà là khám phá codebase: tìm hiểu "cái gì ở đâu, ai gọi ai, đã có những gì".

Risk Analysis — Rủi ro & Mitigation
🧭
"CBM là GPS, không phải autopilot"
GPS cho bạn biết đường đi tổng quan — nhưng bạn vẫn phải nhìn đường khi lái.
Claude dùng CBM để biết "nên tìm ở đâu" — nhưng PHẢI Read/Grep file thực trước khi sửa code.
Rule: CBM informs exploration, never replaces it.
Context Injection
Medium
Stale Data: baseline.json cũ 5 PRs → Claude tin service đã bị xóa vẫn còn → import error
Overconfidence: Claude giảm exploration vì "đã biết" → miss edge cases thực tế
Thiếu logic: Summary chỉ chứa cấu trúc — không chứa business rules, validation, tenant isolation details
✅ Mitigation
• Summary PHẢI có timestamp + commit hash
• Chỉ chứa "what exists" — KHÔNG chứa "how it works"
• Disclaimer: "ALWAYS Read actual files before modifying"
• Post-merge hook auto-regenerate
MCP Server
Medium-High
Server down: MCP crash → Claude retry 3x × 30s = 90s wasted trước khi fallback
Static-only analysis: CBM không hiểu dynamic imports (importlib) → impact report thiếu callers → miss regression
Cross-stack blind: Python AST only → không thấy FE TypeScript impact → BE change break FE
Security: MCP đọc toàn bộ codebase → nếu bị exploit = source code leak
✅ Mitigation
• Response có confidence field: "static-only"
• Cross-verify bằng Grep cho critical paths
• MCP chỉ chạy localhost — không expose network
• Timeout 5s/query, graceful fallback
Quantify Tracking
Low
Sample bias: 5 PRs có thể toàn task đơn giản hoặc phức tạp → kết quả không đại diện
Hawthorne effect: Claude biết đang bị đo → tự giảm exploration → số liệu đẹp nhưng quality giảm
Attribution: Savings có thể do nhiều yếu tố (dev kinh nghiệm, task quen) — khó tách biệt CBM vs khác
✅ Mitigation
• Control group: 2-3 PRs KHÔNG dùng CBM
• Normalize: tokens_per_SP thay vì tokens_per_PR
• Cùng task type: feature FE+BE để apple-to-apple
• Không share tracking với Claude trong session

CTO Warning
"Claude tự tin sai còn nguy hiểm hơn Claude không biết."
Khi Claude nhận summary sẵn, nó có xu hướng giảm exploration — tin summary thay vì verify. Nếu summary sai 1 chi tiết, Claude lan truyền lỗi đó vào code mà không tự phát hiện. Đây là rủi ro #1 của Context Injection.
