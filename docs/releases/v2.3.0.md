# Release v2.3.0 — Claude Code Integration

**Release date:** 2026-04-19
**PyPI:** https://pypi.org/project/codebase-map/2.3.0/
**Install:** `pipx install codebase-map==2.3.0`
**Previous:** v2.2.1

## Highlights

First release with native Claude Code integration. CBM is now a first-class
dependency graph tool for AI coding workflows.

- **Claude Code Skill** — auto-invoked when the user asks about impact,
  callers, dependency graphs, refactor blast radius, API catalogs, or PR
  diffs
- **3 slash commands** — `/cbm-onboard`, `/cbm-impact`, `/cbm-diff`
- **30-second install** — new QUICKSTART guide with one-liner `curl` setup

## What's new

### `integrations/claude-code/skills/codebase-map/SKILL.md`

A Claude Code Skill file that teaches Claude when and how to use CBM.
Auto-invokes on 11 trigger phrases including "impact", "callers", "who
calls", "dependency graph", "onboard this repo", "what breaks if I change
X", "blast radius", "API endpoints in this project", "diff vs main",
"before/after refactor", "is this safe to delete".

### Slash commands

- `/cbm-onboard` — Generate graph + deliver a 5-bullet onboarding summary
  for an unfamiliar repo. Auto-creates `codebase-map.yaml` if missing.
- `/cbm-impact <name>` — Blast-radius analysis at `--depth 2`. Flags
  high-risk refactors at >50 affected nodes and suggests a split strategy.
- `/cbm-diff [base]` — Generate a Markdown PR-body section. Prefers the
  dual-snapshot diff when a `baseline` snapshot is present, falls back to
  git-diff mode otherwise.

### QUICKSTART guide

`integrations/claude-code/QUICKSTART.md` — 30-second setup for Claude Code
users, including a single `curl` one-liner that installs the skill and all
three slash commands into `~/.claude/`.

## Upgrade notes

From v2.2.1:

- No breaking changes. `pipx upgrade codebase-map` works cleanly.
- To use the new Claude integration, copy the skill + commands into
  `~/.claude/` per the QUICKSTART guide.
- Existing `codebase-map.yaml` configs are fully compatible.
