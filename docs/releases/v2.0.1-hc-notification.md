## Có gì mới

- **Layer classifier đã fix triệt để:** Trên repo HC, số node ở layer "unknown" đã giảm từ **173/1565 (11.1%) → 0/1565 (0.00%)**. Toàn bộ 1565 node giờ đã được phân loại đúng layer.
- **Executive view hiển thị coverage thật:** Health bar ở Executive view giờ đọc dữ liệu coverage thật từ `coverage.xml`/JSON. Nếu chưa có coverage thì fallback về proxy như cũ (có tooltip chú thích).
- **API Catalog có empty state thân thiện:** Project Python-only không có HTTP route sẽ hiển thị placeholder rõ ràng thay vì trang trắng.
- **CLI mới `generate --diff <ref>`:** Một lệnh duy nhất sinh ra HTML kèm PR Diff view (không cần chạy `codebase-map diff` rồi `generate` 2 bước nữa).
- **Polish:** TS parser hooks, HTML escape cho API domain header, a11y aria-hidden cho empty-state icon.

## Cách upgrade

```bash
pipx upgrade codebase-map
# hoặc
pip install -U git+https://github.com/hypercommerce-vn/codebase-map.git
```

## Cách dùng flag mới

```bash
# Sinh graph + bake luôn PR diff so với main
codebase-map generate --diff main

# Vẫn dùng cách cũ được nếu muốn
codebase-map generate -c codebase-map.yaml
codebase-map diff main
```

## HC team test gì giúp mình

1. **Re-run trên HC repo:** Xác nhận unknown layer = 0, tất cả 1565 node đều có layer hợp lệ.
2. **Mở Executive view:** Health bar phải hiển thị % coverage thật (nếu repo có `coverage.xml`/JSON), hoặc fallback proxy nếu không có.
3. **API Catalog:** Mở tab API Catalog, nếu repo không có route HTTP thì check xem placeholder có hiển thị copy đúng không.
4. **CLI `--diff`:** Thử `codebase-map generate --diff main` trên 1 branch feature, mở HTML và check tab PR Diff đã có data sẵn.
5. **Báo lại bất kỳ regression nào** (đặc biệt là layer assignment khác lạ trên các module quen thuộc).
