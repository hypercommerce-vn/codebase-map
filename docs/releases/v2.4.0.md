# Release v2.4.0 — MCP Server

**Release date:** 2026-04-20
**PyPI:** https://pypi.org/project/codebase-map/2.4.0/ (pending tag push)
**Install (MCP):** `pipx install "codebase-map[mcp]"` → run `cbm-mcp`
**Previous:** v2.3.0

## Highlights

Codebase Map now ships a **Model Context Protocol (MCP) server** with 5
auto-invoke tools. Claude Desktop, Claude Code, and Cowork can call CBM
directly — no slash commands, no manual `codebase-map` CLI invocations.

- **5 MCP tools** — `cbm_query`, `cbm_search`, `cbm_impact`,
  `cbm_snapshot_diff`, `cbm_api_catalog`
- **Graph cache 40× speedup** — first call parses `graph.json`, subsequent
  calls hit an in-process cache (cold ~3 ms · hot <1 ms on a 200-node graph)
- **`cbm-mcp` entry point** — `pipx install "codebase-map[mcp]"` adds it to
  your `PATH`
- **Single-package strategy** — no separate `codebase-map-mcp` package to
  maintain (MCP server lives in the same distribution, behind an optional
  extra)

## Install + Claude Desktop config

```bash
pipx install "codebase-map[mcp]"
cbm-mcp --help   # blocks on stdio — that's the MCP protocol
```

Add to `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "codebase-map": {
      "command": "cbm-mcp"
    }
  }
}
```

Restart Claude Desktop. Ask: "What's the blast radius of `QueryEngine`?"

## What's new

### `mcp_server/` package

The MCP server + 5 tool modules + a `GraphCache` singleton with mtime-based
invalidation. Stdio transport via the official `mcp>=1.27.0` SDK.

### Tool reference

| Tool | Use when user asks |
|------|-------------------|
| `cbm_query` | "where is X defined", "details of X" |
| `cbm_search` | "find X", "search for X" |
| `cbm_impact` | "blast radius of X", "what breaks if I change X" |
| `cbm_snapshot_diff` | "before/after diff", "PR impact report" |
| `cbm_api_catalog` | "list API endpoints" |

### Entry point

`pyproject.toml` adds `cbm-mcp = "mcp_server.server:main"` alongside the
existing `codebase-map` CLI entry. Both ship in the same wheel.

## Upgrade notes

From v2.3.0:

- No breaking changes. `pipx upgrade codebase-map` works cleanly.
- MCP is an optional extra — install with `[mcp]` to pull `mcp>=1.27.0`.
- Claude Code Skill + slash commands (shipped in v2.3.0) remain supported;
  MCP is an additional path, not a replacement.
