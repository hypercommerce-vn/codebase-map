Issues detected during review-gate. Non-blocking but tracked for follow-up.

KMP-ISSUE-01: vault-format-spec schema extras vs architecture.md spec-drift
vault-format-spec.md adds metadata_json column and AUTOINCREMENT to patterns table that are not in architecture.md §4.1. Also adds idx_patterns_confidence index. These are forward-compatible improvements but create drift between public spec and internal architecture doc.
Source: PR #51 review-gate (CTO, Dim C) · Priority: Low · Action: Reconcile architecture.md in M1 or add "Spec Extensions" note
KMP-ISSUE-02: Dead register_parser call in e2e test test-clarity
test_runtime_full_pipeline calls runtime.register_parser(parser) but LearnerRuntime.run() never uses _parsers. The parser is used externally to generate evidence fed into StubVault. Cosmetic: remove the line or add a comment.
Source: PR #51 review-gate (CTO, Dim A) · Priority: Low · Action: Clean up in Day 5 or M1
KMP-ISSUE-03: Index/column minor diffs (IF NOT EXISTS, DEFAULT) spec-drift
vault-format-spec uses CREATE INDEX IF NOT EXISTS and success BOOLEAN NOT NULL DEFAULT 1 while architecture.md has bare CREATE INDEX and success BOOLEAN. The spec versions are better practice for production.
Source: PR #51 review-gate (CTO, Dim C) · Priority: Low · Action: Update architecture.md to match spec in M1
KMP-ISSUE-04: File naming discrepancy (hello_parser.py vs parsers.py) doc-drift
Implementation uses hello_parser.py / hello_learner.py while architecture.md §5 Reference vertical shows parsers.py / learners.py. README correctly documents reality (actual file names). Architecture doc needs update.
Source: PR #51 review-gate (CTO, Dim B) · Priority: Low · Action: Update architecture.md §5 in M1
KMP-ISSUE-05: Learner cluster() methods ~130-140 LOC refactor
Both LayerLearner.cluster() and GitOwnershipLearner.cluster() are ~130-140 lines. Each cluster type should be extracted into a private sub-method for readability and testability.
Source: PR #58 + #59 review-gate (CTO, Dim B) · Priority: Low · Action: Refactor in M2
KMP-ISSUE-06: _parse_git_log lacks mock-based unit test test-gap
_parse_git_log() runs subprocess.run against real git repo. No mock test exists to verify parsing logic in isolation. Should add unittest.mock.patch test in M2 to cover output parsing without requiring a git repo.
Source: PR #59 review-gate (CTO) · Priority: Low · Action: Add mock test in M2 test coverage task
KMP-ISSUE-07: _extract_domain_from_path duplicated in 2 learners DRY
_extract_domain_from_path() exists in both layer_learner.py and git_ownership_learner.py with near-identical logic. Should extract to a shared verticals/codebase/utils.py module.
Source: PR #59 review-gate (CTO, Dim B) · Priority: Low · Action: Extract shared utils when adding 4th learner or in M2
KMP-ISSUE-08: vault.py growing to ~600 LOC refactor
CodebaseVault now has ~600 LOC with store/query methods for nodes, edges, ownership, patterns, and snapshots. Consider splitting query methods into a separate queries.py module to keep vault.py focused on init/open/schema.
Source: PR #57 + #59 review-gate (CTO, Dim B) · Priority: Low · Action: Split in M2 when adding MCP query tools
KMP-ISSUE-09: _step_learn() accesses runtime._learners (private) encapsulation
BootstrapOrchestrator._step_learn() accesses runtime._learners (private attribute) to iterate learner results. Should expose a public property or method on LearnerRuntime for learner enumeration.
Source: PR #60 review-gate (CTO) · Priority: Low · Action: Add public API in M2
KMP-ISSUE-10: Node/edge extraction in _step_parse() could be helper refactor
_step_parse() does AST parsing + node/edge extraction + stats collection in one method (~40 LOC). The node/edge extraction logic could be extracted to a _extract_graph_data() helper for clarity and testability.
Source: PR #60 review-gate (CTO) · Priority: Low · Action: Refactor in M2
KMP-ISSUE-11: Duplicate patterns across snapshots (4x repeat) ✓ Fixed PR #63
44 patterns output but only ~11 unique. Each pattern repeated 4x with accumulating counts (PascalCase: 46→92→138→184). Root cause: patterns_generator or commit_pattern() doesn't deduplicate — learners process all snapshots and commit duplicate entries.
Source: CEO dogfood review (CTO analysis) · Priority: P0 BLOCKING · Action: Fix in D9
KMP-ISSUE-12: Git author shows filename instead of author name ✓ Fixed PR #63
GitOwnershipLearner outputs "BRIEF.md owns 80.6%" — BRIEF.md is a filename, not an author. _parse_git_log() is extracting the wrong field from git log output.
Source: CEO dogfood review (CTO analysis) · Priority: P0 BLOCKING · Action: Fix in D9
KMP-ISSUE-13: File concentration mixes absolute and relative paths ✓ Fixed PR #63
layers › file concentration shows both knowledge_memory/verticals/codebase: 110 nodes and /Users/pro/Projects/codebase-map/knowledge_memory/verticals/codebase: 330 nodes. Paths should be normalized to relative-only.
Source: CEO dogfood review (CTO analysis) · Priority: P1 · Action: Fix in M2 or D9 if time
KMP-ISSUE-14: NamingLearner counts __init__ as naming violation P1 · false-positive
13 "violations" in snake_case analysis are all __init__ dunder methods (e.g. APICatalog.__init__). Classifier sees uppercase from class name prefix → "mixed". Dunder methods (__init__, __repr__, __str__) should be excluded from naming convention analysis.
Source: CTO review of quick-wins.md · Priority: P1 · Action: Fix in M2 — exclude dunder methods from naming check
KMP-ISSUE-15: Layer compliance 100% misleading when unknown >50% P2 · UX
"Layer compliance: 100%" when 93% of nodes are unknown layer is technically correct (detected = stored) but misleading. When unknown >50%, title should show "Layer classification: X% recognized" instead of "compliance 100%".
Source: CTO review of quick-wins.md · Priority: P2 · Action: Improve in M2 — add recognized-layer awareness
KMP-ISSUE-16: Depth analysis action too generic for normal nesting P3 · UX
"Avg depth 3.5 — Review deeply nested modules" but depth 3-4 is normal for Python packages. When avg_depth ≤ 4, action should be "Normal nesting — no action needed" instead of "Review deeply nested modules".
Source: CTO review of quick-wins.md · Priority: P3 · Action: Improve in M2 — threshold-based action text
