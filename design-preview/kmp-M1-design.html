Designer note
5-step visual rhythm: [N/5] bold header + description + progress bar + result + timing. Each learner gets its own sub-progress line during step 3. Final summary uses horizontal rule separator + green checkmark + total stats. Error state: If a learner crashes, show ⚠ NamingLearner failed (skipped) — see .knowledge-memory/error.log and continue.
CTO constraint
Performance SLA: ≤5 min for 100K LOC. If scan detects >200 files, show estimated time. Graceful shutdown: Ctrl+C during any step must save partial state — no corrupt vault.db. Learner isolation: One learner crash must NOT abort pipeline — log error + continue remaining learners. Confidence threshold: Only patterns with ≥60% confidence are committed (configurable in config.yaml).

Designer note
3 sections: Structure (5) + Patterns (3) + Risks (2) = 10 insights total. Confidence score right-aligned: green ≥80%, yellow 60-79%, red = risk items. Each insight: numbered + bold title + one-line detail + evidence path + confidence. Exactly 20 lines max for CLI output. Full details in quick-wins.md file.
CTO constraint
Insight categories: 5 structure + 3 patterns + 2 risks (fixed ratio). Confidence color: ≥80% green, 60-79% yellow, below 60% not shown (filtered by threshold). Evidence required: Every insight must cite at least 1 file path. Deterministic: Same codebase → same quick wins (no random sampling).

G Design Decisions from PO + CTO + Designer Meeting

3-way meeting 10/04/2026. Decisions locked for M1 implementation.
#DecisionOwnerRationale
D-M1-01 CLI-first, no HTML in M1 PO Ship fast. Terminal output + Markdown files. HTML dashboard deferred to M3.
D-M1-02 10 Quick Wins fixed ratio: 5+3+2 PO + CTO Structure insights most valuable for first use. Risks always surface last. Consistent output across projects.
D-M1-03 Rich CLI via rich library Designer Progress bars, colored text, tables. NO_COLOR=1 fallback. pip dependency already light.
D-M1-04 Confidence threshold: 60% default CTO Balances signal vs noise. Configurable in config.yaml. Patterns below threshold not committed.
D-M1-05 Learner isolation: crash one, continue rest CTO Pipeline resilience. Error logged to .knowledge-memory/error.log. Partial results still useful.
D-M1-06 Snapshot rotation: keep 5 CTO Vault disk footprint predictable. Older snapshots auto-deleted on bootstrap. Configurable.
D-M1-07 Ctrl+C saves partial state CTO + Designer UX: never lose work. --resume flag continues from last completed learner.
D-M1-08 Output files are standard Markdown PO Renderable everywhere: GitHub, VSCode, any editor. No custom format. Git-trackable.
