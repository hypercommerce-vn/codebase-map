CM-S3-06

TypeScript Parser — Architecture (BE)

5 SP BE only

Parser thứ 2 sau Python. Dùng tree-sitter-typescript (WASM) để extract functions/classes/methods/imports/routes từ .ts/.tsx files. Không có UI mới — output merge vào cùng Graph model để Graph View hiển thị được.

Mockup 06 — Parser architecture
codebase_map/parsers/ ├── base.py ← BaseParser (unchanged) ├── python_parser.py ← Existing (v1.0) └── typescript_parser.py ← NEW (CM-S3-06) │ ▼ tree-sitter-typescript (WASM bundled) │ ▼ Extract: • FunctionDeclaration → NodeType.FUNCTION • ClassDeclaration → NodeType.CLASS • MethodDefinition → NodeType.METHOD • ImportDeclaration → Edge(IMPORT) • CallExpression → Edge(CALL) • Decorators @Controller → NodeType.ROUTE │ ▼ Same Node/Edge dataclasses │ ▼ GraphBuilder merges Python + TypeScript
File extParserFramework detect
.pypython_parser (AST)FastAPI / Flask / Django / Celery
.ts, .tsxtypescript_parser (tree-sitter)NestJS / Express / Next.js API routes
.js, .jsxtypescript_parser (permissive mode)Express / Next.js
  • Parity test: parse HC frontend (if TS) → ≥95% functions captured
  • Performance: 10k LOC TS file < 2s
  • Offline: tree-sitter WASM bundled, no npm/node runtime required
  • Graceful fallback: syntax error → skip file + warning, không crash
